# Sesion 2026-05-04 — CU17 Fase 6 (Recordatorios automaticos)

## Objetivo
Implementar backend de recordatorios y notificaciones automaticas sin Celery (estrategia cron-friendly), manteniendo `/api/` y sin tocar frontend.

## Cambios principales
- Nuevo modulo `backend/apps/notificaciones/automatizaciones/` con:
  - modelos: `ReglaRecordatorio`, `TareaRecordatorioProgramada`, `LogEjecucionRecordatorio`
  - serializers y servicio de procesamiento
  - viewsets y rutas DRF (`reglas`, `tareas`, `logs`)
  - admin
  - migracion inicial `0001_initial.py`
  - management command `procesar_recordatorios`
- Integracion en `config/settings.py` con `apps.notificaciones.automatizaciones`.
- Integracion de rutas dentro de `apps/notificaciones/urls.py`.
- Tests minimos en `backend/apps/notificaciones/automatizaciones/tests/test_automatizaciones.py`.

## Seguridad y permisos
- Mutaciones restringidas a `IsAdministrativoOrAdmin` para reglas y acciones de tareas.
- Lectura autenticada para reglas/tareas/logs.
- Bitacora en create/update/delete de reglas y en generacion de tareas.

## Validacion ejecutada
- `pytest apps/notificaciones/automatizaciones/tests/test_automatizaciones.py`
  - Error real de entorno: `ModuleNotFoundError: No module named 'django'`
  - Warning: `Unknown config option: DJANGO_SETTINGS_MODULE`
- `python -m py_compile ...` sobre archivos nuevos/modificados del modulo: OK.
